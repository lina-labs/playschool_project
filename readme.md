# Overview

This project is a story-driven web game that combines frontend interface design with backend script processing.
The frontend was built with **HTML, CSS, and JavaScript** to create the game interface and user interaction flow.
On the backend, **Python** was used to process story scripts stored in CSV files, segment character dialogues by scene or narrative structure,
and store them in **MongoDB** for flexible retrieval and management.

To **support dynamic scene transitions**,
the project uses **Flask** together with **Jinja2** for server-side rendering,
allowing different HTML pages to be generated according to story progression.
This architecture helps make the game flow more dynamic and scalable.
